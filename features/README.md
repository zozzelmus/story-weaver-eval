# features/ — held-out test inputs

One file per eval case: the **raw requirement** (what goes into the tool). Keep it in
pre-ticket voice — a requirement, not tickets.

- Filename = the CASE-ID, e.g. `0007-saml-sso.md`.
- Each one needs a matching `reference-tickets/<CASE-ID>.md` (the human answer).
- Aim for ~15 cases for a directional read.

`EXAMPLE-0001-bulk-export.md` is a worked example — replace it with real features.
